# Issue #50: Document and wire `setup-worktree` hook

## Complexity
medium

## Context / Problem
`wade` supports a `hooks.post_worktree_create` config key that executes a user-supplied script after every worktree is created (`wade work start`). The hook is fully implemented in `work_service._bootstrap_worktree`, loaded by `config/loader.py`, and validated by `check_service._validate_hooks_section`. However:

1. **README has zero mention** of the `hooks` section or `post_worktree_create`.
2. **`wade init` does not emit** a `hooks` section (not even commented-out) in the generated `.wade.yml`, so users have no way to discover the feature.
3. **`docs/dev/architecture.md`** only shows the hook key in a config snippet without explaining semantics, execution environment, or failure behavior.

## Proposed Solution
1. **Document the hooks section** in the README under a new "Worktree hooks" subsection (or expand existing config reference).
2. **Include a commented-out `hooks` block** in the `.wade.yml` generated by `wade init` (via `_write_config` in `init_service.py`) so users see the option immediately.
3. **Expand `docs/dev/architecture.md`** to explain hook semantics: execution directory (worktree root), env vars passed (if any), timeout (60 s), silent success / warning on failure.
4. **Add a minimal sample script** at `templates/setup-worktree.sh.example` that users can copy to `scripts/setup-worktree.sh` in their project.
