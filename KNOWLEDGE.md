## 2b4d5319 | 2026-04-11 | implementation | Issue #272

When knowledge commands run from a linked git worktree, load_config() finds
the .wade.yml copied to the worktree and sets project_root to the worktree
directory. The fix: detect the main checkout using git worktree list
--porcelain and redirect all knowledge reads/writes there.
_canonical_project_root() handles expected operational errors (ImportError,
GitError, OSError) and falls back to project_root; unexpected exceptions
propagate so real bugs are not silently swallowed. Apply at the top of service
functions (append_knowledge, get_annotated_knowledge) and in CLI commands
that resolve paths directly (resolve_canonical_knowledge_path wrapper).
