## db750a19 | 2026-03-31 | implementation | Issue #248

Gemini CLI hooks format: .gemini/settings.json expects hooks as an object keyed by event name (e.g. {"hooks": {"BeforeTool": [{"matcher": ".*", "hooks": [{"type": "command", "command": "..."}]}]}}) — NOT a flat array. upsert_hook_entry supports this via ensure_path=["hooks", "BeforeTool"]. Gemini CLI deprecated --allowed-tools flag; use Policy Engine at .gemini/policies/*.toml with [[rule]] blocks instead (no stdlib TOML writer needed — manual string formatting is fine since [[rule]] format is trivial).
