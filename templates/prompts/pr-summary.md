Write a PR summary for Issue #{issue_number}: {issue_title}

The summary should be written to /tmp/PR-SUMMARY-{issue_number}.md and include:

1. What was accomplished (high-level, 2-3 sentences)
2. Key changes (bullet list)
3. Technical decisions (if any non-obvious choices were made)
4. Testing done (how you verified the changes)
